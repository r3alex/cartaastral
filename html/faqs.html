Tetrabyblos is "broken" - files are missing or don't work

 

Depending of the place of purchase, you will be receiving a zipped file, with different names or a different structure. The important thing is that you should install in your Wordpress website the file named Tetrabyblos.zip - do not unzip this file please. This file must be uploaded like it is when you are making the installation.

Also, do not attemp to install the entire file you receive from the purchase. Once unzipped, you should have a directory called "Documentation" (or even a zipped file with the same name) and the Tetrabyblos.zip file, that you need for a correct installation. Just upload, wait for the installation process, and activate.

Tetrabyblos is not working: the cities auto-complete doesn't work, or the hour format tabs are not properly presented, etc.

(Steps for detecting and solving the problem)

The reason basically nothing is working, is because (most likely) there are minor or even major disconfigurations in the javascript code of your webpage, namely (and very important) the basic libraries (like jquery.js, etc.) that are not loaded for some reason. This is not, in the majority of the cases, the plugin fault. These libraries should be loaded even if the plugin was not installed.

Tetrabyblos is not a perfect plugin (no plugin is), but it has worked so far with the majority of the users, and the failures cases are most due to other plugins or even template themes you might be using in your website.

Do you have, for example, any javascript optimizer that you might have installed, or anything of the type? I had users which had this kind of problem, and they were using these optimizers that, for example, compress the code, or remove potentially dangerous routines that are not (false positives). These, sometimes, mess with the code and necessary routines, even not enabling - for instance - a simple jquery.js library to load (which it should by default in Wordpress). So the auto-complete of the cities won't work, or the tabs sections will show unformated, etc., because the proper javascript is not correctly loaded and/or Tetrabyblos code doesn't find the necessary libraries.

Please try to verify that first. Once identified the problem, the troublemaker should be deactivated.

I would advise the following procedure (it's what I do when I have this sort of problem):

1 - deactivate all the plugins (non essential).
2 - change to a basic theme
3 - activate for example Tetrabyblos
4 - see if it runs - it must, otherwise you will have to make a new installation (in this case, save the languages settings)
5 - then, one by one, activate the plugins
6 - test Tetrabyblos each time
7 - when the present scenario appears, you found the problematic plugin (unistall it and try to find a similar one that doens't generate those problems).

If you feel stuck, please contact me and I'll gladly try to help.

The file is "too big"

Very rare problem

This problem usually happens because your webhost provider restricted the size of the files that can be uploaded to your website. The most common warning expression isthat the file “exceeds the maximum upload size for this site”.

Please contact the host tech support, and expose the situation and they will surely solve the problem without questioning. Tetrabyblos does not need much space, but the max. filesize allowed by the webhost is sometimes too small because usually the files used in your website, like images, are also small and prevents that some of your users might try to upload very big files.

If the space is yours, you can try to edit the php.ini if you feel sure enough. By default, the maximum upload file size for PHP scripts is set to 128 megabytes. However, you may have to change these limits. For example, you can set a lower limit to prevent users from uploading large files to your site. To do this, change the upload_max_filesize and post_max_size directives. To ensure that file uploads work correctly, the post_max_size directive should be a little larger than the upload_max_filesize. For example, the following settings demonstrate how to set a file upload limit to 20 megabytes:

upload_max_filesize = 20M
post_max_size = 21M

"Strange" php warnings and/or errors

Very rare problem

Sometimes, other errors or warnings show up on the admin panel (or in the pages) telling you that are errors related to "fopen()", "fgets()", or "feof()" commands, among others.

The problem is undoubtedly in the general configuration of the server, with regard to php. If the space is yours, you can try to edit the php.ini file (if you feef sure - otherwise ask your host tech support) and search for allow_url_fopen. This should be defined as:

allow_url_fopen = 1

If you do not have access to php.ini, you will need to ask your host server for assistance.

There may be other small details that you will need to ask to be activated if other errors appear. This is common on Wordpress host servers, which tend to limit installation resources and disable common options. You may need to change the minimum file size accepted, memory, etc.

Still, these are very rare issues.
